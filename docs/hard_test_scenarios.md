# Hard Test Scenarios — Edge-Case Stress Tests

30 сценариев, ориентированных на сложные случаи, связанные с ограничениями задачи 3D Pallet Packing.
Расположены в `rl_solver/scenarios/hard_scenarios.py`.

---

## Категория 1: Weight Limit Edge Cases (1–5)

Тесты на граничные условия по весу паллеты.

| # | ID | Описание | Паллета | Коробки | Суть |
|---|-----|---------|---------|---------|------|
| 1 | `hard_weight_exact` | Точное попадание в лимит веса | 1200×800×1800, 100 кг | 4× Heavy block (25 кг) | Суммарный вес = 100 кг = max_weight. Все должны влезть |
| 2 | `hard_weight_overflow_1` | Переполнение на 1 коробку | 1200×800×1800, 50 кг | 6× Dense cube (10 кг) | 60 кг > 50 кг — 1 коробка обязана остаться unplaced |
| 3 | `hard_weight_one_heavy` | Один тяжёлый среди лёгких | 1200×800×1800, 200 кг | 50× Light (0.5 кг) + 1× Anchor (180 кг) | Anchor занимает 90% бюджета веса. Остаётся 20 кг на 50 лёгких |
| 4 | `hard_weight_uniform_cutoff` | Равномерный обрез по весу | 1200×800×600, 120 кг | 12× Uniform heavy (15 кг) | 12 × 15 = 180 кг > 120 кг → максимум 8 коробок |
| 5 | `hard_weight_greedy_trap` | Жадная ловушка по весу | 1200×800×1800, 80 кг | 4× Big light (2 кг) + 5× Small heavy (18 кг) | Жадный по объёму возьмёт большие лёгкие, но маленькие тяжёлые быстро исчерпают вес |

## Использование

```bash
# Генерация JSON файлов
python -m rl_solver.scenarios.hard_scenarios

# Запуск бенчмарка на всех сценариях
python -m rl_solver.rl_benchmark --agents ga,sa --include-original

# Тренировка RL агентов
python -m rl_solver.train --agent all --episodes 200
```

## Ключевые метрики

Каждый сценарий проверяется через `validator.py` с расчётом:
- **Volume Utilization** (50%) — плотность укладки
- **Item Coverage** (30%) — доля размещённых коробок
- **Fragility Compliance** (10%) — отсутствие нарушений хрупкости
- **Execution Time** (10%) — скорость решения
